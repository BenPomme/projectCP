Finding CORS Settings on Google Cloud Storage

Step 1: Access Bucket List

Go to Google Cloud Console Storage Browser and make sure you're in the correct project (kingcp-a0065).

Step 2: Select Your Bucket

Click on the bucket name kingcp-a0065.appspot.com from the list to enter the bucket's details page.

Step 3: Find CORS Settings

Look for one of the following:

  • A tab labeled "Configuration" at the top of the page
  • A side panel with a section for "CORS"
  • A three-dot menu (⋮) in the top-right corner, which might contain an option for "Edit CORS settings" or "Bucket settings"
  • Check both the "Permissions" tab and scroll all the way down for a CORS section

Alternative Method: Via "Bucket Details"

Sometimes CORS settings are hidden behind an additional menu:

  1. Find a "Bucket details" button near the top (sometimes represented by an information icon ℹ️ or gear icon ⚙️)
  2. Look for CORS configuration in the detailed information that appears

Alternative Method: Direct URL

Try accessing this direct URL:

https://console.cloud.google.com/storage/browser/kingcp-a0065.appspot.com;tab=cors

CORS Configuration to Add

When you find the CORS settings, add this configuration:

[
  {
    "origin": ["https://benpomme.github.io"],
    "method": ["GET", "POST", "PUT", "DELETE", "HEAD"],
    "maxAgeSeconds": 3600,
    "responseHeader": ["Content-Type", "Content-Disposition", "Content-Length"]
  }
]

Can't Find CORS Settings?

If you still can't find the CORS settings, here are some other options:

  1. Install Google Cloud SDK on your machine and use the gsutil command line
  2. Try using the Firebase Extension for CORS configuration: Firebase Storage CORS Extension
  3. Contact Google Cloud support if you have a paid plan
